# Story 1.2: Autenticação com Supabase Auth

| Campo | Valor |
|-------|-------|

## User Story

**Como** usuário de marca D2C,
**Quero** criar uma conta e fazer login de forma segura,
**Para que** eu possa acessar a plataforma com meus dados protegidos.

## Acceptance Criteria

- [x] 1. Tela de registro com: nome da marca, email, senha — cria conta no Supabase Auth
- [x] 2. Tela de login com email/senha
- [x] 3. Opção de login via magic link (email)
- [x] 4. Middleware de proteção de rotas: usuário não autenticado é redirecionado para login
- [x] 5. Botão de logout funcional
- [x] 6. Tabela `tenants` criada no Supabase com campos: id, name, created_at
- [x] 7. Ao registrar, um tenant é criado automaticamente e vinculado ao usuário
- [x] 8. Tabela `tenant_users` com: user_id, tenant_id, role (admin), created_at
- [x] 9. RLS policy básica: usuário só acessa dados do próprio tenant

## Technical Notes

- Supabase Auth: email/senha + magic link
- Trigger `handle_new_user()` já definido na migration 00001
- Middleware Next.js em `middleware.ts`
- RLS via `get_user_tenant_id()`
