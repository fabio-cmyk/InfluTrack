# Story 2.2: Identidade Visual e Upload de Assets

| Campo | Valor |
|-------|-------|

| **Status** | Ready |

## User Story

**Como** gestor de marca,
**Quero** cadastrar a identidade visual (logo, cores, tipografia),
**Para que** o brand asset esteja completo e acessivel para referencia.

## Acceptance Criteria

- [x] 1. Secao de identidade visual na pagina de Branding (abaixo ou em tab separada do formulario de identidade textual)
- [x] 2. Upload de logotipo (PNG, SVG, JPG) via Supabase Storage com preview da imagem
- [x] 3. Paleta de cores: ate 6 cores com input de hex code e preview visual (quadrado colorido)
- [x] 4. Tipografia: campos para fonte primaria e fonte secundaria (nome da fonte)
- [x] 5. Dados salvos na tabela `brand_visual_identity` vinculada ao tenant
- [x] 6. Assets de imagem armazenados no Supabase Storage com bucket por tenant
- [x] 7. RLS no Storage: apenas membros do tenant acessam os arquivos (ja configurado na migration 00002 para tabela)

## Technical Notes

- Tabela `brand_visual_identity` ja definida na migration `00002_create_brand_assets.sql`
- Campos: logo_url (TEXT), color_palette (JSONB - array de {name, hex}), primary_font (TEXT), secondary_font (TEXT)
- Relacao 1:1 com tenant (UNIQUE constraint em tenant_id)
- RLS policies ja configuradas: SELECT para todos do tenant, INSERT/UPDATE com `branding.edit`
- `color_palette` formato: `[{"name": "Primary", "hex": "#6C63FF"}, ...]` - max 6 cores
- Supabase Storage: criar bucket `brand-assets` com policy por tenant
- Upload flow: client-side file selection -> server action upload para Storage -> salvar URL na tabela
- Usar upsert pattern igual Story 2.1
