# Story 2.1: Cadastro de Identidade de Marca

| Campo | Valor |
|-------|-------|

| **Status** | Ready |

## User Story

**Como** gestor de marca,
**Quero** cadastrar as informacoes da minha marca (missao, visao, valores, tom de voz, publico-alvo),
**Para que** a plataforma conheca minha marca e possa avaliar compatibilidade com influencers.

## Acceptance Criteria

- [x] 1. Pagina "Branding" acessivel via sidebar (rota /branding ja existe como stub)
- [x] 2. Formulario com campos: nome da marca, missao, visao, valores (lista editavel), tom de voz (textarea), publico-alvo (textarea)
- [x] 3. Campos adicionais: dores do cliente (lista), beneficios do produto (lista), diferenciais competitivos (lista), palavras-chave da marca (tags)
- [x] 4. Dados salvos na tabela `brand_assets` vinculada ao tenant
- [x] 5. Formulario com botao salvar com feedback visual (toast/mensagem de sucesso)
- [x] 6. RLS: apenas usuarios do tenant podem ler/escrever (ja configurado na migration 00002)

## Technical Notes

- Tabela `brand_assets` ja definida na migration `00002_create_brand_assets.sql`
- Campos: brand_name, mission, vision, values (TEXT[]), tone_of_voice, target_audience, customer_pain_points (TEXT[]), product_benefits (TEXT[]), competitive_differentiators (TEXT[]), brand_keywords (TEXT[])
- Relacao 1:1 com tenant (UNIQUE constraint em tenant_id)
- RLS policies ja configuradas: SELECT para todos do tenant, INSERT/UPDATE com `branding.edit`
- Trigger `update_updated_at()` ja configurado
- Usar upsert pattern: INSERT se nao existe, UPDATE se ja existe
- Pagina stub ja existe em `src/app/(dashboard)/branding/page.tsx`
