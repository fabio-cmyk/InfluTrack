## User Story

**Como** gestor de marca,
**Quero** cadastrar e gerenciar influencers na plataforma,
**Para que** eu tenha uma base centralizada de todos os influencers que trabalho.

## Acceptance Criteria

- [x] 1. Pagina "Influencers" com tabela listando todos os influencers do tenant
- [x] 2. Colunas: nome, handle principal, nicho, seguidores, cupom, nr campanhas
- [x] 3. Busca por nome/handle e filtro por nicho
- [x] 4. Botao "Adicionar Influencer" abre formulario com: nome, email, telefone, Instagram handle, TikTok handle, YouTube handle, cidade, estado, nicho/categoria
- [x] 5. Campo de cupom exclusivo: input manual, validacao de unicidade dentro do tenant
- [x] 6. Edicao de influencer em pagina dedicada
- [x] 7. Soft delete (arquivar) - nao remove dados historicos
- [x] 8. Tabela `influencers` com todos os campos + tenant_id + timestamps (migration 00003)
- [x] 9. RLS: isolamento por tenant (ja configurado na migration 00003)

## Technical Notes

- Tabela `influencers` na migration `00003_create_influencers.sql`
- Campos: name, email, phone, instagram_handle, tiktok_handle, youtube_handle, city, state, niche, coupon_code, avatar_url, is_archived
- Coupon normalizado para UPPERCASE via trigger `normalize_coupon_code()`
- UNIQUE constraint em (tenant_id, coupon_code)
- Soft delete via `is_archived` boolean
- RLS: influencers.view para SELECT, influencers.edit para INSERT/UPDATE
- Pagina stub em `src/app/(dashboard)/influencers/page.tsx`
